Situación de Aprendizaje: ¿Funciona correctamente? Diseño y ejecución de pruebas

Sesión 1. Introducción a pruebas

Objetivo: Comprender los conceptos de validación y verificación del software
Contenidos:

Durante todo el proceso de desarrollo de software es necesario realizar un conjunto de pruebas que permitan verificar la calidad del software, que el software es correcto y cumple con las especificaciones impuestas por el usuario. 
Las pruebas se integran en las diferentes fases del ciclo de vida del software.

Verificación: Comprobación que un sistema o parte de un sistema, cumple con las condiciones impuestas.

  1. Se centra en el proceso
  2. Se debe realizar durante todo el ciclo de desarrollo
  3. Busca errores en documentación, diseño de código.
  4. Responde a ¿Estamos construyendo el programa correctamente?

Validación: Evaluación del sistema o de uno de sus componentes, para determinar si satisface los requisitos especificados.

  1. Se centra en el producto
  2. Se realiza al final del ciclo de desarrollo, el producto esta listo para ser entregado
  3. Busca errores de comportamiento o funcionalidad
  4. Responde a: ¿Estamos construyendo el programa correcto?

Actividad: Debate en clase, ¿Qué puede pasar si una red social tiene errores?
