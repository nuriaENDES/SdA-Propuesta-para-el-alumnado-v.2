Diccionario de términos

Texto

  • Testing: proceso de evaluación del software
  • Prueba unitaria: prueba de una función o clase
  • Caso de prueba: situación concreta a evaluar
  • Bug: error en el software
  • Regresión: comprobar que algo sigue funcionando después de modificar
  • Validación: Responde a ¿Se construye el programa correcto?
  • Verificación: Responde a ¿Se construye el programa correctamente?
  • Pruebas funcionales: Prueba si las salidas que devuelve la aplicación son las esperadas: corresponde a pruebas de caja negra
  • Pruebas estructurales: Verifican la estructura interna de cada componente de la aplicación: corresponde a pruebas de caja blanca
  • Pruebas unitarias: Prueban cada unidad o modulo con el objetivo de eliminar errores en la interfaz y en la lógica interna
  • Pruebas de integración: Pruebas para comprobar como interaccionan los distintos módulos. Comprueba como funcionan juntos
  • Pruebas de validación: Validan el sistema a través de dos modalidades: Pruebas alfa y pruebas beta
  • Pruebas de sistema: Consiste en un conjunto de pruebas que tiene como objetivo ejercitar de forma profunda  el software
